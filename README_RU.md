# Автоматизация развертывания Django в продакшн

Этот репозиторий содержит набор Bash-скриптов, предназначенных для автоматизации развертывания, управления и защиты Django-приложений на серверах Ubuntu/Debian с использованием Nginx и Gunicorn.

## 🚀 Обзор скриптов

| Скрипт | Описание |
| :--- | :--- |
| `deploy.sh` | Выполняет начальную настройку сервера, клонирует репозиторий, настраивает виртуальное окружение, устанавливает Gunicorn (socket и service) и настраивает Nginx. |
| `secure.sh` | Устанавливает Certbot и настраивает SSL-сертификаты Let's Encrypt для работы через HTTPS. |
| `update.sh` | Загружает последнюю версию кода из GitHub, обновляет зависимости, запускает миграции и перезапускает сервисы. |
| `cleanup.sh` | Отменяет все изменения, останавливает службы, удаляет конфигурационные файлы и директорию проекта. |

---

## 📋 Предварительные требования

- Чистый сервер Ubuntu или Debian.
- Пользователь с правами `sudo`.
- Публичный GitHub-репозиторий с вашим Django-проектом.
- Доменное имя, указывающее на IP-адрес вашего сервера (необходимо для `secure.sh`).

## ⚙️ Настройка проекта Django

Перед запуском скрипта `deploy.sh` убедитесь, что ваш файл `settings.py` настроен правильно для работы со статическими файлами и хостами в продакшне:

```python
# settings.py

DEBUG = False  # Установите False для продакшна
ALLOWED_HOSTS = ["mydomain.com", "www.mydomain.com", "your_server_ip"]

STATIC_URL = 'static/'
STATICFILES_DIRS = [BASE_DIR / 'static'] 
STATIC_ROOT = BASE_DIR / 'staticfiles'

MEDIA_URL = 'media/'
MEDIA_ROOT = BASE_DIR / 'media'
```

*Примечание: Конфигурация Nginx в этих скриптах ищет статику именно в директории `staticfiles/` (параметр `STATIC_ROOT`).*

## 🛠️ Инструкции по использованию

### 1. Скопируйте скрипты на сервер
```bash
mkdir -p ~/deploy-scripts && cd ~/deploy-scripts
# Скопируйте файлы скриптов в эту директорию
```

### 2. Сделайте скрипты исполняемыми
```bash
chmod +x deploy.sh secure.sh update.sh cleanup.sh
```

### 3. Запустите развертывание
Запустите скрипт развертывания и следуйте подсказкам (потребуется ввести URL репозитория и домен/IP):
```bash
./deploy.sh
```
*Примечание: В конце скрипт предложит создать суперпользователя Django.*

### 4. Настройка SSL (рекомендуется)
После того как сайт заработает по HTTP, запустите скрипт для настройки HTTPS:
```bash
./secure.sh
```

### 5. Обновление проекта
Каждый раз, когда вы отправляете новый код в GitHub, запускайте этот скрипт для синхронизации:
```bash
./update.sh
```

### 6. Очистка / Удаление
 Если вам нужно удалить проект и все связанные с ним конфигурации:
```bash
./cleanup.sh
```

## 🔍 Подробное описание работы скриптов

### `deploy.sh`
- Обновляет системные пакеты и устанавливает `python3-pip`, `venv`, `nginx` и `git`.
- Клонирует репозиторий и создает виртуальное окружение Python.
- Устанавливает зависимости из `requirements.txt` (или базовый стек, если файл отсутствует).
- Выполняет команды `collectstatic` и `migrate`.
- Создает сокет и системную службу Gunicorn для управления процессами.
- Настраивает блок сервера Nginx в качестве обратного прокси.
- Устанавливает права доступа для пользователя `www-data` и создает папку `media`.

### `secure.sh`
- Устанавливает `certbot` и плагин для Nginx.
- Получает и устанавливает SSL-сертификаты от Let's Encrypt.
- Автоматически перезагружает Nginx для применения настроек HTTPS.

### `update.sh`
- Загружает свежий код через `git pull`.
- Обновляет Python-зависимости и запускает миграции базы данных.
- Перезапускает Gunicorn и Nginx для применения изменений.
